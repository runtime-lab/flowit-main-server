== 개요

Activity와 Notification 응답은 서버가 완성 문장을 내려주는 구조가 아닙니다.

Activity는 사이트 내에서 발생한 활동을 원장 개념으로 기록하는 아카이브입니다. 사용자의 행위와 시스템 상태 변화가 기록 기준이며, 알림 발송 여부가 핵심이 아닙니다.

Notification은 단발성 이벤트에 대해 사용자에게 알림을 전송했거나 전송하려 한 사실의 기록입니다. 전달 대상, 수신자별 확인/읽음 상태, 링크 힌트 같은 delivery 관점이 핵심입니다.

Activity가 Notification을 유발할 수는 있지만 모든 Activity가 Notification 대상이 되는 것은 아니며, Notification은 Activity의 대체 기록이 아닙니다.

서버는 ``actor``, ``target`` 또는 ``subject``, ``action`` 또는 ``type``, ``changes``, ``link`` 같은 의미 부품을 제공하고, 클라이언트는 화면 맥락에 맞게 문장을 조립할 수 있습니다.

이 문서의 문장은 고정 문구 계약이 아닙니다.
응답 필드의 의미를 이해하고 UI에서 일관된 표현을 만들기 위한 조립 예시입니다.

[cols="1,4",role="endpoint-summary"]
|===
|권장 기준 |목록에서는 짧은 요약 문장을 우선 표시하고, 세부 변경값은 펼침 또는 상세 영역에서 보여줍니다.
|문구 자유도 |클라이언트는 서비스 톤, 화면 밀도, 언어 정책에 맞게 문장을 바꿀 수 있습니다.
|계약 기준 |문장 자체가 아니라 응답 필드, enum 값, null 가능성, 링크 힌트가 API 계약입니다.
|시간 값 |``occurredAt``은 Long 타입의 Unix epoch seconds 값입니다.
|===

== Activity 응답 조립

Activity는 워크스페이스 최근 활동 목록의 한 줄을 구성하기 위한 원장성 응답입니다.
``items[]``의 레코드 하나가 기본 렌더링 단위이며, ``changes[]``는 상세 diff 영역에서 사용하는 것을 권장합니다.

=== 응답 예시

[source,json]
----
{
  "id": 7001,
  "occurredAt": 1780920000,
  "domain": "TASK",
  "actor": {
    "memberId": 3001,
    "userId": 1001,
    "displayName": "Actor"
  },
  "target": {
    "type": "TASK",
    "id": 1001,
    "displayName": "Login UI"
  },
  "action": "STATUS_CHANGED",
  "changeCount": 1,
  "changedElements": [
    "STATUS"
  ],
  "changes": [
    {
      "element": "STATUS",
      "from": "TODO",
      "to": "IN_PROGRESS"
    }
  ]
}
----

=== 조립 예시

[cols="1,4",options="header",role="api-table"]
|===
|용도 |예시

|목록 요약
|``actor.displayName`` + "님이 " + ``target.displayName`` + "의 상태를 변경했습니다."

|목록 출력 예
|Actor님이 Login UI의 상태를 변경했습니다.

|상세 diff
|상태: ``TODO`` -> ``IN_PROGRESS``

|보조 정보
|``occurredAt``을 클라이언트 시간 포맷으로 변환해 함께 표시할 수 있습니다.
|===

=== Activity 필드 사용 기준

[cols="1,4",options="header",role="api-table"]
|===
|필드 |조립 기준

|``actor.displayName``
|활동 수행자 이름입니다. 시스템성 이벤트에서는 ``memberId`` 또는 ``userId``가 ``null``일 수 있으므로 이름만 표시 가능한 구조로 처리합니다.

|``target.displayName``
|활동 대상의 표시 이름입니다. ``target.type``이 ``TASK``이면 작업 이름, ``WORKSPACE_MEMBER``이면 워크스페이스 멤버 이름으로 해석합니다.

|``action``
|문장의 핵심 동사 또는 템플릿 선택 기준입니다. 예를 들어 ``STATUS_CHANGED``는 상태 변경, ``JOINED``는 가입, ``REMOVED``는 강제 퇴장 문구를 선택할 수 있습니다.

|``changedElements``
|목록 요약에 사용할 변경 요소 목록입니다. 여러 요소가 변경된 경우 "제목 외 2개 항목을 수정했습니다." 같은 축약 표현에 사용할 수 있습니다.

|``changeCount``
|변경 개수 요약에 사용할 수 있습니다. ``changes.length``와 같은 의미의 요약 필드입니다.

|``changes[]``
|상세 diff에 사용합니다. 기본 목록 문장에 모든 ``from``/``to`` 값을 넣으면 문장이 길어질 수 있으므로 펼침 또는 상세 UI에서 사용하는 것을 권장합니다.
|===

=== Activity 추가 예시

[cols="1,1,4",options="header",role="api-table"]
|===
|action |주요 필드 |조립 예시

|``CREATED``
|``actor``, ``target``
|Actor님이 Login UI 작업을 만들었습니다.

|``MODIFIED``
|``actor``, ``target``, ``changedElements``
|Actor님이 Login UI의 제목 외 2개 항목을 수정했습니다.

|``PROGRESS_CHANGED``
|``actor``, ``target``, ``changes[].from/to``
|Actor님이 Login UI의 진행도를 변경했습니다. 상세: ``20`` -> ``60``

|``JOINED``
|``actor``, ``target``
|Target님이 워크스페이스에 가입했습니다.

|``ROLE_CHANGED``
|``actor``, ``target``, ``changes[].from/to``
|Actor님이 Target님의 역할을 변경했습니다. 상세: ``MEMBER`` -> ``ADMIN``

|``REMOVED``
|``actor``, ``target``
|Actor님이 Target님을 워크스페이스에서 내보냈습니다.

|``WITHDRAWN``
|``actor`` 또는 ``target``
|Target님이 워크스페이스에서 탈퇴했습니다.
|===

== Notification 응답 조립

Notification은 현재 사용자에게 표시할 개인 알림입니다.
목록 API의 ``data.items[]``와 WebSocket payload가 같은 ``NotificationAlertResponse`` 구조를 사용합니다.
Notification의 ``read``와 ``seen``은 활동 원장의 상태가 아니라 수신자별 알림 처리 상태입니다.

=== 응답 예시

[source,json]
----
{
  "id": 101,
  "type": "WORKSPACE_MEMBER_REMOVED",
  "occurredAt": 1782013200,
  "scope": {
    "type": "WORKSPACE",
    "id": 12,
    "name": "Flowit"
  },
  "actor": {
    "type": "USER",
    "id": 34,
    "name": "Actor",
    "profileImageUrl": "/v1/users/34/profile-image"
  },
  "subject": {
    "type": "WORKSPACE_MEMBER",
    "id": 55,
    "name": "Target"
  },
  "changes": [
    {
      "element": "MEMBERSHIP",
      "from": "ACTIVE",
      "to": "REMOVED"
    },
    {
      "element": "ROLE",
      "from": "MEMBER",
      "to": null
    }
  ],
  "link": {
    "type": "WORKSPACE_MEMBERS",
    "workspaceId": 12
  },
  "read": false
}
----

=== 조립 예시

[cols="1,4",options="header",role="api-table"]
|===
|용도 |예시

|목록 요약
|``actor.name`` + "님이 " + ``subject.name`` + "님을 " + ``scope.name`` + " 워크스페이스에서 내보냈습니다."

|목록 출력 예
|Actor님이 Target님을 Flowit 워크스페이스에서 내보냈습니다.

|상세 diff
|멤버십: ``ACTIVE`` -> ``REMOVED`` / 역할: ``MEMBER`` -> 없음

|링크 힌트
|``link.type``이 ``WORKSPACE_MEMBERS``이면 클라이언트가 필요에 따라 멤버 관리 화면 이동을 구성할 수 있도록 ``link.workspaceId``를 함께 제공합니다.
|===

=== Notification 필드 사용 기준

[cols="1,4",options="header",role="api-table"]
|===
|필드 |조립 기준

|``type``
|알림 문장 템플릿의 1차 선택 기준입니다. 같은 워크스페이스 멤버 변경이라도 ``WORKSPACE_MEMBER_REMOVED``와 ``WORKSPACE_ACCESS_REVOKED``는 사용자 관점이 다릅니다.

|``scope``
|알림이 발생한 범위입니다. 현재는 ``WORKSPACE`` 범위이며, ``scope.name``을 문장에 넣어 어느 워크스페이스의 알림인지 보여줄 수 있습니다.

|``actor``
|알림을 발생시킨 주체입니다. 주체가 없는 이벤트를 대비해 ``actor.id``와 ``actor.name``은 ``null`` 가능성을 고려합니다.

|``subject``
|알림 대상입니다. 워크스페이스 멤버 알림에서 ``subject.id``는 ``workspaceMemberId``이며 사용자 식별자가 아닙니다.

|``changes[]``
|알림의 원인이 된 변경 내용입니다. 목록에서는 생략하거나 한 줄로 요약하고, 상세 영역에서 ``element``별 ``from``/``to``를 표시할 수 있습니다.

|``link``
|클릭 또는 후속 이동에 사용할 힌트입니다. ``link.type``이 ``NONE``이면 워크스페이스 내부 화면으로 이동시키지 않습니다.

|``read``
|현재 사용자 기준 읽음 상태입니다. 알림 자체의 전역 상태가 아니라 수신자별 상태입니다.
|===

=== Notification 유형별 예시

[cols="1,1,4",options="header",role="api-table"]
|===
|type |관점 |조립 예시

|``WORKSPACE_MEMBER_JOINED``
|워크스페이스 구성원
|Target님이 Flowit 워크스페이스에 가입했습니다.

|``WORKSPACE_MEMBER_ROLE_CHANGED``
|워크스페이스 구성원
|Actor님이 Target님의 역할을 변경했습니다.

|``WORKSPACE_MEMBER_REMOVED``
|남아 있는 구성원
|Actor님이 Target님을 Flowit 워크스페이스에서 내보냈습니다.

|``WORKSPACE_ACCESS_REVOKED``
|강제 퇴장 당사자
|Flowit 워크스페이스 접근 권한이 회수되었습니다.

|``WORKSPACE_MEMBER_WITHDRAWN``
|워크스페이스 구성원
|Target님이 Flowit 워크스페이스에서 탈퇴했습니다.
|===

== null 및 값 타입 처리

[cols="1,4",options="header",role="api-table"]
|===
|상황 |권장 처리

|``actor``의 식별자 또는 이름이 ``null``
|문장 주어를 "시스템" 또는 화면 정책에 맞는 대체 표현으로 처리합니다. 단, 현재 사용자 이름으로 임의 치환하지 않습니다.

|``changes[].from``이 ``null``
|생성, 가입처럼 이전 값이 없는 변경으로 해석할 수 있습니다. 상세 표시에서는 "없음 -> 새 값" 또는 "새 값으로 설정" 형태를 사용할 수 있습니다.

|``changes[].to``가 ``null``
|삭제, 해제, 퇴장처럼 이후 값이 없는 변경으로 해석할 수 있습니다. 상세 표시에서는 "기존 값 -> 없음" 형태를 사용할 수 있습니다.

|``changes[].from/to``가 객체
|객체 내부 구조를 그대로 문장에 넣지 말고, UI에서 표시할 이름 또는 핵심 식별자를 선택해 표시합니다.

|알 수 없는 enum 값
|클라이언트는 기본 fallback 문장을 두는 것이 좋습니다. 예를 들어 "활동이 업데이트되었습니다.", "새 알림이 도착했습니다."처럼 데이터 손실 없이 표시합니다.
|===
