.표준 API 포맷
====

[cols="28,30,42",role="api-index"]
|===
|문서 |Endpoint |설명

|link:docs-preview.html[문서 미리보기 API]
|`POST /api/docs-preview/projects/{projectId}/tasks`
|커스텀 Spring REST Docs 스니펫의 요청, 응답, 필드 문서 예시를 확인합니다.

|link:api-response.html[공통 응답 포맷]
|성공/에러 응답 envelope
|API 응답의 공통 `success`, `data`, `error`, `extensions` 구조를 설명합니다.

|link:integration-notes.html[Integration Notes]
|최신/누적 통합 변경 기록
|빠르게 변하는 API 통합 변경 사항을 최신 note와 버전별 아카이브로 설명합니다.

|link:version-policy.html[Version Policy]
|`major.minor.patch-stage`
|버전 증가 기준과 `chamber`, `volt`, `neon` stage 정책을 설명합니다.

|link:error-codes.html[에러 코드]
|`error.code`
|클라이언트 다국어 처리와 에러 분기에 사용하는 에러 코드 목록을 설명합니다.

|link:enum-reference.html[Enum Reference]
|공통 enum 값
|응답 필드에서 참조하는 enum 값과 의미를 설명합니다.
|===

[%collapsible]
.인증/인가 API
====

[cols="28,30,42",role="api-index"]
|===
|문서 |Endpoint |설명

|link:auth-login.html[로그인 API]
|`POST /v1/public/auth/login`
|JWT access token 발급과 HttpOnly refresh token 쿠키 전달 방식을 설명합니다.

|link:auth-token-refresh.html[토큰 재발급 API]
|`POST /v1/public/auth/refresh`
|Refresh token 쿠키를 사용한 access token 재발급 방식을 설명합니다.

|link:auth-logout.html[로그아웃 API]
|`POST /v1/public/auth/logout`
|Refresh token 로그아웃 처리와 쿠키 만료 응답을 설명합니다.
|===
