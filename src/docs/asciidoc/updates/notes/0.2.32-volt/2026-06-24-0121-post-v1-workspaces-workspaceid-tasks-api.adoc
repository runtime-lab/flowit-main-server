Integration Note

Added:: ``POST /v1/workspaces/{workspaceId}/tasks`` 작업 생성 API의 요청에 옵셔널 필드를 추가했습니다.
+
* 요청 필드 ``progress``가 추가되었습니다.
* ``progress``를 생략하거나 ``null``로 전달하면 기존과 같이 ``0``을 적용합니다.
* ``progress`` 값이 있는 경우 ``0``부터 ``100``까지의 정수만 허용합니다.
* 이 변경은 ``breaking change``가 아닙니다. 기존 요청 계약을 그대로 사용 할 수 있습니다.
Carryover:: 개인 알림 API가 추가되었습니다.
+
* ``GET /v1/notifications`` 는 현재 사용자의 알림 목록, 전체 개수, 읽지 않은 알림 개수, 아직 확인하지 않은 알림 개수를 반환합니다.
* ``PATCH /v1/notifications/seen`` 은 알림 오버레이가 열렸음을 서버에 알려 아직 확인하지 않은 알림을 확인 처리합니다.
* ``PATCH /v1/notifications/read-all`` 은 현재 사용자의 읽지 않은 알림을 모두 읽음 처리합니다.
* 확인 / 알림은 서로 다른 상태로 확인은 ``오버레이를 펼친 행위`` / 읽음은 ``사용자의 명시적인 상태 변경 요청``으로 취급합니다.
Carryover:: 알림 WebSocket 문서와 구독 계약이 추가되었습니다.
+
* 클라이언트는 ``/ws`` 로 STOMP WebSocket을 연결하고 ``/user/queue/notifications`` 를 구독합니다.
* STOMP ``CONNECT`` native header의 ``Authorization: Bearer {accessToken}`` 값으로 사용자를 인증합니다.
* WebSocket payload는 알림 목록 API의 ``data.items[]`` 와 같은 ``NotificationAlertResponse`` 구조를 사용합니다.
Carryover:: 워크스페이스 멤버 변경 알림이 알림/소켓 파일럿 대상으로 추가되었습니다.
+
* 멤버 가입, 역할 변경, 강제 퇴장, 자진 탈퇴 이벤트가 개인 알림으로 저장됩니다.
* 강제 퇴장 시 기존 워크스페이스 구성원에게는 ``WORKSPACE_MEMBER_REMOVED`` 알림이 전달됩니다.
* 강제 퇴장 당사자에게는 ``WORKSPACE_ACCESS_REVOKED`` 알림이 별도로 전달됩니다.
Carryover:: 워크스페이스 활동 기록에서 알림 생성 흐름을 분리했습니다.
+
* 활동 기록은 기존대로 남기고, 알림 발행 요청은 outbox를 통해 후속 처리됩니다.
* 알림은 접속 중인 사용자에게 WebSocket으로 추가 전달되지만, 기준 상태는 알림 조회 API를 통해 복구합니다.
* WebSocket 수신만으로 ``read`` 또는 ``seen`` 상태는 변경되지 않습니다.
Carryover:: 기존 ``/v1/users/me`` 응답 계약은 아직 변경하지 않았습니다.
