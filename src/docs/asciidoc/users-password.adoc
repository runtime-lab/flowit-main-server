== 현재 사용자 비밀번호 변경

[cols="1,4",role="endpoint-summary"]
|===
|메서드 |`PATCH`
|경로 |`/v1/users/me/password`
|API 상태 |[.badge.stability.stable]#Stable#
|목적 |현재 비밀번호를 검증한 뒤 인증된 사용자의 비밀번호를 변경합니다.
|인증 |JWT access token이 필요합니다.
|대상 사용자 |요청 본문이나 경로에서 사용자 ID를 받지 않고, access token의 subject로만 수정 대상을 결정합니다.
|현재 비밀번호 검증 |`currentPassword`가 기존 비밀번호 해시와 일치해야 변경됩니다.
|토큰 영향 |성공 시 사용자 `token_version`이 증가합니다. 이전 버전의 access token은 보호 API에서 거부되며, refresh token 쿠키는 만료 처리됩니다.
|클라이언트 처리 |성공 응답을 받으면 저장 중인 access token을 폐기하고 로그인 화면으로 이동해야 합니다.
|===
